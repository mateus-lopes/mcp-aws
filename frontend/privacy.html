Privacy Policy

Last updated: June 29, 2026

MCP AWS lets authenticated users connect their own AWS credentials and manage AWS resources through a web dashboard, API, or GPT Action. This policy explains what data is handled by the service.

Information We Collect

  • Account information, such as email address and hashed password.
  • AWS credential records voluntarily added by the user, including access key ID, secret access key, optional session token, label, and AWS region.
  • OAuth authorization codes and access tokens used to connect a GPT Action to the user's MCP AWS account.

How We Use Information

Data is used to authenticate users, associate each GPT Action request with the correct MCP AWS user, and perform AWS operations only with the AWS credentials that user configured.

GPT Action Access

When OAuth is enabled, ChatGPT sends an access token to MCP AWS. The API uses that token to identify the authenticated MCP AWS user. The GPT Action does not receive another user's AWS credentials through this service.

AWS Credentials

AWS credentials are stored so the API can call AWS services on behalf of the authenticated user. Users should provide least-privilege AWS credentials and remove credentials that are no longer needed.

Data Sharing

MCP AWS sends requests to AWS APIs as required to perform user-requested actions. The service does not sell user data.

User Control

Users can remove AWS credential records from the dashboard or API. Removing a credential prevents future AWS calls using that stored credential.

Security

Passwords are hashed before storage. OAuth tokens are stored as hashes. AWS credentials should be treated as sensitive secrets and access should be limited to trusted deployments.

Contact

For questions about this policy, contact the MCP AWS administrator responsible for this deployment.
